Arrow Functions

Intro

In ES6, the common way of describing a function would become something like

[1, 2, 3, 4].map(num => {

var multiplier = 2 + num

return num * multiplier

})

It looks very similar to a normal function declaration, excpet for the "=>", and that precisely is the topic of the chapter, the Arrow Function

The changes can be pointed as,

There is no longer any need to specify the "function" keyword.

The function is bound to its lexical scope.

LEXICAL SCOPE (BLOCK SCOPE)

The scope of anything is said to be lexical if its scope is limited to the enclosing { and }, that is its enclosing box

(In javascript, a variable declared anywhere is hoisted and the scope of the variable becomes to that of the enclosing class/function)

What this means that all the tricks like

var self = this

var that = this

.bind(this)

are no longer necessary when using arrow functions.

... unless you want to use the scope set during events. In such cases you have to use the "function" syntax.

ONE LINE STATEMENTS

The arrow function is particularly useful when writing small functions like

[1, 2, 3].map(function (num) { return num * 2 })

[1, 2, 3].map(num => num * 2)

[1, 2, 3, 4].map((num, index) => num * 2 + index)

The first example, being a larger function is better off written in the previous syntax. (as you also get to name the function which is better in most of the cases)
