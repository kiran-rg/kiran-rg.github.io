Modules

Intorduction

To obtain modules in JavaScript we have used systems like RequireJS, CommonJS etc. But now with ES6 we have a built in module system.

Strict Mode

In the ES6 module system, strict mode is turned on by default. It’s just a stricter version of the language that disallows lots of bad parts of the language.

Exports

Default Export

export default 1

export default NaN

export default 'foo'

export default { foo: 'bar' }

export default ['foo', 'bar']

export default function foo () {}

export statements can only be placed at the top level in ES6 modules – even if the method they’re in

would be immediately invoked when loading the module.

function foo () {

export default 'bar' // SyntaxError

}

foo()

Named Exports

export var foo = 'bar'

export var baz = 'ponyfoo'

Bindings, Not Values

// foo.js

let a = {a: 1};

function mutateA() {

a = {b: 2};

}

export a;

export mutateA;

// bar.js

import {a, mutateA} from foo;

console.log(a); // {a: 1}

mutateA();

console.log(a); // {b: 2}

Exporting Lists

var foo = 'ponyfoo'

var bar = 'baz'

export { foo, bar }

If you’d like to export something with a different name, you can use the export { foo as bar }

export { foo as default, bar }

Imports

Import Default Exports

import 'lodash'

import _ from 'lodash'

Import Named Exports

import {map, reduce} from 'lodash'

import {cloneDeep as clone, map} from 'lodash'

import {default, map} from 'lodash'

import {default as _, map} from 'lodash'

import _, {map} from 'lodash'

Import All

import * as _ from 'lodash'
